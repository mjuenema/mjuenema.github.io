Pycom SiPy and Sigfox

Sigfox

Over the last few days I started experimenting with the Sigfox Low Power Wide Area Network (LPWAN). As a LPWAN, Sigfox is aimed at the Internet-of-Things (IoT) and trades throughput for long distance communications and long battery life.

Sigfox operates in the ISM radio band. Because of ISM band regulations the transmission duty cycle is limited to 1% equalling at most 140 messages per day per device. Most applications will probably send a lot fewer messages per day in order to save power.

The payload of Sigfox message is 96 bits (12 bytes) for the uplink and 64 bits (8 bytes) for downlink messages but a downlink message will only be send if a device polls for one.

The Sigfox protocol does not define any media access mechanism. A device simply transmits each message three times on different frequencies, expecting that at least one basestation withing range (which can be tens of kilometres away) will receive it successfully. The basestation will then forward the message to a Sigfox backend server for processing.

Pycom SiPy

As hardware I decided to buy a Pycom SiPy micro-controller for the simple reason that it can be programmed with MicroPython. This makes it an ideal platform for quick proto-typing. I added the Pycom Expansion Board and the Antenna Kit.

Pycom also offer other interesting expansion boards but for simply familiarising yourself with the SigFox service those are not needed.
