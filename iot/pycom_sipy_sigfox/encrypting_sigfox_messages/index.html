Encrypting Sigfox messages

Encrypting Sigfox messages poses some unique challenges.

  • Block ciphers algorithms like AES normally operate on data of fixed length. In case of AES this is 128 bits which is larger than the 96 bits payload of Sigfox.
  • Block Ciphers can be operated in stream mode but require an "initialisation vector" (IV) which must also be transmitted. The IV, at least in the case of AES, must be of the same size as the cipher's block size, i.e. 128 bits, it again exceeds the payload size of a Sigfox message. The only possible work-around is to transmit the encrypted data in two Sigfox messages which is inefficient.

There is a whole new research domain of Lightweigth Cryptography for the Internet of Things. The eSTREAM project resulted in a suite of ciphers aimed at resource constrained environments.

Another very interesting candidate is the Speck cipher. What makes this cipher so appealing to Sigfox is that it offers small and variable length block sizes of 32, 48, 64 and 96 bits. Calvin McCoy implemented Speck in Python which I used for encrypting Sigfox messages on the Pycom Sipy.
